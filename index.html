创建代收订单，用户支付后资金进入商户账户

notify_url String 是 异步通知地址 http://apis.nrny.com/index/test/notify

sign String 是 签名 A1B2C3D4E5F6G7H8I9J0K1L2M3N4O5P6

代付接口

POST /repay/unifiedOrder

创建代付订单，从商户账户向指定账户转账

重要提醒： 代付接口会直接从您的账户扣款，请确保接收方信息准确无误。

请求参数

参数名 类型 必填 说明 示例值
merchant_id String 是 商户号 10001
amount Decimal 是 代付金额，保留2位小数 100.00
currency String 是 货币类型 BRL
channel_code String 是 代付通道编码 pix_transfer
merchant_order_no String 是 商户订单号，唯一 TRANSFER20240101001
notify_url String 是 异步通知地址 https://your-site.com/transfer-notify
subject String 是 代付说明 佣金发放
extra_params Object 是 收款人信息 见下表
timestamp Integer 是 时间戳 1640995200
sign_type String 是 签名类型 md5
sign String 是 签名 A1B2C3D4...

extra_params 收款人信息

参数名 类型 必填 说明 示例值
name String 是 收款人姓名 João Silva
CPF String 否 巴西个人税号 12345678901
PHONE String 否 手机号码 +5511999999999
EMAIL String 否 邮箱地址 joao@example.com
EVP String 否 PIX随机密钥 abc123-def456-ghi789
收款人信息说明： CPF、PHONE、EMAIL、EVP 四个字段至少填写一个，系统会按优先级自动选择支付方式。

请求示例

POST /repay/unifiedOrder HTTP/1.1 Host: api.yourpay.com Content-Type: application/json { "merchant_id": "10001", "amount": "100.00", "currency": "BRL", "channel_code": "pix_transfer", "merchant_order_no": "TRANSFER20240101001", "notify_url": "https://your-site.com/transfer-notify", "subject": "佣金发放", "extra_params": { "name": "João Silva", "CPF": "12345678901", "PHONE": "+5511999999999", "EMAIL": "joao@example.com" }, "timestamp": 1640995200, "sign_type": "md5", "sign": "B2C3D4E5F6G7H8I9J0K1L2M3N4O5P6Q7" }

响应参数

参数名 类型 说明 示例值
code Integer 状态码，0表示成功 0
msg String 返回信息 success
data Object 返回数据 -
data.order_no String 系统订单号 SYS20240101002
data.status String 订单状态 1

响应示例

{ "code": 0, "msg": "success", "data": { "order_no": "SYS20240101002", "status": "1" } }

订单状态说明

状态值 状态名称（代收/代付） 说明

3 已退款 / 已支付 订单已退款
-1 失败 / 失败 支付失败
-2 已过期 订单已过期

代收回调通知

重要说明： 当代收订单状态发生变化时，系统会向您提供的notify_url发送异步通知。

通知机制

  • 通知方式：HTTP POST请求
  • 数据格式：JSON
  • 重试机制：失败后会重试，间隔时间递增
  • 超时时间：30秒
  • 成功标识：返回字符串"success"

通知参数

参数名 类型 说明 示例值

status String 订单状态 成功=2，失败=-1 2
pay_time String 支付完成时间 1755555448
timestamp String 通知时间戳 1755555448
sign String 签名 B8536FFC8A20EF2848E6395FA4D142C3

通知示例

POST /your-notify-url HTTP/1.1 Host: your-site.com Content-Type: application/json { "merchant_id": "M2501082278", "order_no": "P202508190323213489c6b6b", "merchant_order_no": "TEST1755544994315", "amount": "100.00", "real_amount": "100.00", "status": "2", "pay_time": "1755555448", "timestamp": "1755555448", "sign": "B8536FFC8A20EF2848E6395FA4D142C3" }

签名验证

收到通知后，请务必验证签名确保数据安全：

// PHP示例 function verifySign($params, $secret) { $sign = $params['sign']; unset($params['sign']); ksort($params); $string = ''; foreach ($params as $key => $value) { $string .= $key . '=' . $value . '&'; } $string .= 'key=' . $secret; return strtoupper(md5($string)) === $sign; }

响应要求

重要： 收到通知后，请返回字符串"success"表示处理成功，否则系统会认为通知失败并重试。

凭证状态说明

状态值 状态名称 说明
available 可用 凭证已生成，可以下载
generating 生成中 凭证正在生成，请稍后查询
unavailable 不可用 订单状态不支持生成凭证
注意事项：
  • 只有已支付成功的订单才能生成凭证
  • 凭证下载链接有效期为24小时
  • 凭证为PDF格式，包含完整的交易信息
  • 每个订单的凭证只能生成一次

签名算法

所有接口请求都需要进行签名验证，签名算法如下：

1. 将所有请求参数（除sign外）按参数名ASCII码升序排序 2. 将排序后的参数按 key=value&key=value 格式拼接 3. 在拼接字符串末尾加上 &key=商户密钥 4. 对拼接后的字符串进行MD5加密，转为大写

签名示例

// 原始参数 { "merchant_id": "10001", "amount": "100.00", "currency": "BRL", "timestamp": "1640995200" } // 排序后拼接 amount=100.00¤cy=BRL&merchant_id=10001×tamp=1640995200&key=YOUR_SECRET_KEY // MD5加密后 sign = MD5("amount=100.00¤cy=BRL&merchant_id=10001×tamp=1640995200&key=YOUR_SECRET_KEY").toUpperCase()

时间戳验证

为防止重放攻击，系统会验证请求时间戳，要求：

  • timestamp 参数为10位Unix时间戳
  • 请求时间与服务器时间差不超过5分钟
  • 建议每次请求使用当前时间戳
